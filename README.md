## Project Structure

```
jbrowse-launch/
├── jbrowse2/          # JBrowse 2 web app build
│   └── config.json    # Active JBrowse configuration (do not edit directly)
├── server/            # Static file server (TypeScript/Bun)
├── config.json        # JBrowse config; generated by just add-assembly/add-track
└── justfile           # Task runner
```

## Start

The static server serves both the JBrowse app (`jbrowse2/`) and local data files (`test_data/`) from a single origin (`http://localhost:3000`).

**1. Clone and enter the repo**

```sh
git clone <repo-url>
cd jbrowse-launch
```

**2. Install dependencies**

```sh
bun install
```

**3. Create a local data directory**

```sh
mkdir test_data
```

Place your FASTA and GFF3 files here. If you use a different directory name, update `assets_dir` in the justfile.

**4. Add a FASTA assembly**

```sh
just add-assembly test_data/canonical_core.fa
```

Indexes the file with `samtools faidx` and registers the assembly in `config.json`.

**5. Add a GFF3 track** *(optional)*

```sh
just add-track test_data/my_genes.gff3
```

Sorts, compresses, indexes, and registers the track in `config.json`.

**6. Start the dev server**

```sh
just dev
```

Loads `config.json` into `jbrowse2/config.json` and starts the server. Open `http://localhost:3000`.

## Adding Data Files

#### FASTA

```sh
just add-assembly <filename.fa>
```

#### GFF3

```sh
just add-track <filename.gff3>
```

## Configuration

| File | Purpose |
|---|---|
| `config.json` | JBrowse config; generated by `just add-assembly` / `just add-track` |
| `jbrowse2/config.json` | Active config loaded by the app; overwritten by `just dev` |

`just load-config` copies `config.json` → `jbrowse2/config.json`.

## Static Server

The server is a minimal Bun HTTP server (`server/main.ts`) with automatic MIME type detection.

```sh
# Direct usage
bun serve <root> <assets> [-p <port>]

# Via just
just serve             # defaults: jbrowse2/, test_data/, port 3000
just serve port=8080   # custom port
```

## Recreating the JBrowse Build

Downloads and unpacks a fresh JBrowse 2 web app into `jbrowse2/` and applies `config.json`.

```sh
just create
```
