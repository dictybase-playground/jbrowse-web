# dictybase/jbrowse-web

This project contains static server that serves a build of the `jbrowse` app

## Requirements

- [bun](https://bun.sh/)
- [just 1.4+](https://github.com/casey/just)
- [samtools](https://www.htslib.org/)

## Project Structure

```
jbrowse-launch/
├── jbrowse2/          # JBrowse 2 web app build
│   └── config.json    # Active JBrowse configuration (do not edit directly)
├── server/            # Static file server (TypeScript/Bun)
├── config.local.json  # Local JBrowse config (gitignored, developer-specific)
└── justfile           # Task runner
```

## Development

Each developer maintains their own `config.local.json` that points to the data files they have available locally. This file is not committed — it is generated by the `just add-assembly` command and loaded into the JBrowse app at dev time.

The static server (`server/index.ts`) serves both the JBrowse app (`jbrowse2/`) and local data files (`test_data/`) from a single origin (`http://localhost:3000`).

### Quick Start

**1. Install Bun dependencies**

```sh
bun install
```

**2. Create local Jbrowse data directory**

This directory will be served by the static server and should contain your data files (FASTA, GFF3, etc.). You can name it `test_data/` or something else.

*note*: If you name it something other than `test_data/`, you will need to adjust the `serve` command in the package.json accordingly.

Example:
```sh
mkdir test_data
```

**3. Place a FASTA file into data directory**

Place your FASTA file in data directory you just created, then run:

**3. Add a FASTA assembly to local config**

Example:
```sh
just add-assembly test_data/canonical_core.fa
```

This indexes the file with `samtools faidx` (producing a `.fa.fai` file) and registers the assembly in `config.local.json`, pointing to `http://localhost:3000/<data_directory>/<fasta_file>`.

**4. Load the config file into the Jbrowse application directory**

```sh
just load-config-local
```
This copies the  `config.local.json` file into `jbrowse2/config.json`, which is the config file loaded by the JBrowse app.

**5. Start the dev server**

```sh
just serve
```

This starts the static server. Open `http://localhost:3000` in your browser.

### Adding Data Files

#### FASTA

```sh
just add-assembly <filename.fa>
```

The file must be in `test_data/`. The command indexes it and adds the assembly to `config.local.json`.

#### GFF3

Index the file manually, then add the track to `config.local.json` by hand or via the JBrowse CLI:

```sh
bgzip test_data/annotations.gff3
tabix test_data/annotations.gff3.gz
bun run jbrowse add-track http://localhost:3000/test_data/annotations.gff3.gz \
  --out config.local.json
```

### Static Server

The dev server is a minimal Bun HTTP server in `server/main.ts`. It serves files from a given directory with automatic MIME type detection.

```sh
# Run directly
bun server/index.ts <directory>

# With a custom port
PORT=8080 bun server/index.ts <directory>
```

The `just dev` recipe starts it pointed at the project root so both `jbrowse2/` and `test_data/` are accessible under the same origin.

### Configuration

| File | Purpose |
|---|---|
| `config.local.json` | Developer-local JBrowse config; generated by `just add-assembly` |
| `jbrowse2/config.json` | Active config loaded by the JBrowse app; overwritten by `just dev` |
| `config.json` | Shared/production config committed to the repo |

`just load-config-local` copies `config.local.json` → `jbrowse2/config.json`.
`just load-config` copies `config.json` → `jbrowse2/config.json`.

### Recreating the JBrowse Build

```sh
just create
```

Downloads and unpacks a fresh JBrowse 2 web app into `jbrowse2/`, then loads `config.json`. Only one JBrowse version is kept at a time.
